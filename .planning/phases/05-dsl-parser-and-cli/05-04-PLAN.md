<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process args/stdin/files -> CLI | Untrusted user input enters CLI orchestration. |
| CLI -> local filesystem | CLI writes user-requested output files. |
| diagnostics -> terminal/automation | Untrusted DSL values may appear in errors. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Denial of Service | `readStdin`, `readInputFile` | mitigate | Enforce `MAX_INPUT_BYTES = 1_000_000` for stdin/file reads and parser input. |
| T-05-02 | Tampering | parsed DSL pipeline | transfer | Use Wave 2/3 parser and normalization safeguards; CLI only passes source text and options to `renderDiagramDsl`. |
| T-05-03 | Tampering | `writeFileAtomic` | mitigate | Temp-file-then-rename after successful export; tests assert failed run preserves existing file content. |
| T-05-04 | Tampering | `--format` option | mitigate | Pass raw format into `resolveOutputFormat`; unsupported formats return non-zero and no output write. |
| T-05-05 | Spoofing | human and JSON diagnostics | mitigate | JSON diagnostics are JSON-encoded; human diagnostics are plain one-line fields without source snippet echo or ANSI color. |
</threat_model>
