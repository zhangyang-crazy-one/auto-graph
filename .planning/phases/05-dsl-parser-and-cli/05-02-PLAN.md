    target: api

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DSL text -> parser | Untrusted YAML/JSON crosses into process memory. |
| parsed object -> schema | Parsed plain objects must not become runtime IR by spread. |
| parser diagnostics -> machine consumers | Error payloads must be deterministic and injection-resistant. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Denial of Service | `parseDiagramDsl` | mitigate | Enforce default `maxBytes = 1_000_000` before YAML/JSON parsing and test `parse.input.too-large`. |
| T-05-02 | Tampering | `schema.ts` and parsed objects | mitigate | Validate through Zod and return a typed DSL value; do not spread parsed plain objects into Intent/Normalized IR in this wave. |
| T-05-03 | Tampering | output writes | accept | No file output is implemented in this wave; Wave 4 owns atomic write mitigation. |
| T-05-04 | Tampering | `output.format` validation | mitigate | Zod enum accepts only `svg` and `excalidraw`; deferred `drawio`, `mermaid`, and `ascii` must fail validation. |
| T-05-05 | Spoofing | diagnostic output | mitigate | Diagnostics include layer/code/path and are sorted deterministically; do not include arbitrary multiline DSL source snippets in human messages. |
</threat_model>
