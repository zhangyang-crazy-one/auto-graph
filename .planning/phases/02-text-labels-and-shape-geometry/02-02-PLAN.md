Create label contracts:
```typescript
export interface LabelFitOptions {
	font: TextStyleOptions;
	padding: Insets | number;
	minSize?: Partial<Size>;
	maxWidth?: number;
	overflow?: "allow" | "diagnose" | "truncate";
}

export interface LabelLineLayout {
	text: string;
	box: Box;
	baselineY: number;
	width: number;
	lineIndex: number;
	sourceStart?: TextCursor;
	sourceEnd?: TextCursor;
}

export interface LabelLayout {
	text: string;
	box: Box;
	contentBox: Box;
	naturalSize: Size;
	fittedSize: Size;
	padding: Insets;
	font: TextStyleOptions;
	lineHeight: number;
	lines: LabelLineLayout[];
	overflow: {
		horizontal: boolean;
		vertical: boolean;
		truncated: boolean;
	};
	diagnostics: Diagnostic[];
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller -> fitLabel | Public callers pass text, font, padding, minSize, maxWidth, and overflow mode. |
| labels -> TextMeasurer | Label fitting depends on a measurement backend supplied by caller or default setup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | `fitLabel`, `LabelFitter` | mitigate | Reject non-finite, negative, or impossible sizing inputs before computing boxes. |
| T-02-06 | Denial of Service | `fitLabel` wrapping path | mitigate | Use deterministic linear layout data from `TextMeasurer`; do not add recursive fitting or renderer calls. |
| T-02-07 | Repudiation | Label overflow diagnostics | mitigate | Emit stable diagnostic codes such as `label.overflow.horizontal` and `label.overflow.vertical` for recoverable overflow states. |
| T-02-08 | Information Disclosure | Label APIs | accept | Label text is caller-supplied and returned to caller; no storage, network, or secret access is introduced. |
| T-02-09 | Elevation of Privilege | Renderer creep | mitigate | Add grep verification and acceptance criteria forbidding renderer-native output in labels. |
</threat_model>
